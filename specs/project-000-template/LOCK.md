# LOCK.md — 文件锁定清单

> **说明**：本文件记录当前被锁定的文件。锁定意味着在对应 Sprint 进行中，这些文件不应被修改，  
> 以防止并发变更导致冲突。`lock-check.py` 会在 git commit 时自动检查。  
>
> **锁定时机**：Sprint 开始时，将本 Sprint 涉及的核心文件加入锁定表。  
> **解锁时机**：Sprint 结束归档后，从锁定表中移除，或通过解锁申请流程提前解锁。

---

## 锁定规则说明

### 哪些文件应该被锁定？

- 数据库 Model 文件（Schema 变更影响大）
- 核心 Service 层文件（正在被某个 Sprint 重写时）
- 配置文件（`config.py`、`.env.example`）
- 数据库 Migration 文件（永久锁定，只能新增）
- 公共工具库（`utils/`，变更会影响所有依赖方）

### 哪些文件不需要锁定？

- 测试文件（低风险，鼓励频繁修改）
- 文档文件（`*.md`）
- 单独的新功能文件（不影响已有代码时）

### lock-check.py 的检测方式

`lock-check.py` 读取本文件的**已锁定文件表**，与 `git diff --cached` 对比。  
支持精确路径匹配和 glob 模式匹配（如 `src/models/*.py`）。
