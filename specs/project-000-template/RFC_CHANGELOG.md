# RFC_CHANGELOG.md — 需求变更历史

> **说明**：本文件记录所有已归档的 RFC 变更。  
> 每次执行 `python3 governance/apply-and-archive.py <project-id> <ch-id> archive` 后，  
> 脚本会自动在此文件末尾追加一条记录。  
>
> **重要**：归档后需手动将 `delta.md` 中的变更内容合并回 `RFC.md`，  
> 并将 REQ 条目的 `状态` 字段更新为对应状态。  
>
> 条目格式说明：
> - `CH-ID` — 变更提案编号，如 CH-001
> - `变更标题` — 一句话描述做了什么
> - `涉及需求` — 修改了哪些 REQ-NNN（ADD/MODIFY/DELETE）
> - `归档时间` — 何时完成归档
> - `实施结果` — 是否完整实施，有无遗留
